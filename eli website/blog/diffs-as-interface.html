Diffs as an Interface

2026-01-13 • Eli

Most “coding assistants” paste code. That’s convenient until it isn’t—because the interface is ambiguous. Where should the code go? What changed? What broke?

Text is not an API. Diffs are.

Why diffs matter

Eli communicates changes as diffs so every edit is:

- explicit (what file, what lines)
- reviewable (preview vs apply)
- reversible (undo)

Guardrails without killing momentum

Eli supports approval modes so you can choose the tradeoff:

/plan  → ask before running commands and applying diffs
/bot   → run commands automatically, ask before diffs
/yolo  → auto approvals (full speed)

The punchline

Once you constrain edits to diffs and add undo, you can let smaller models act aggressively without fear. The harness provides the safety net.
