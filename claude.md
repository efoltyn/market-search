# Eli - Rust CLI for Quantitative Research

## What is Eli?

**Eli is a terminal AI agent that works like an AI quant intern.**

It's a Rust CLI similar to Claude Code, Codex, and Cursor - but specialized for financial analysis. Give it a research question, and it autonomously:
- Fetches market data via native Rust tools (Yahoo Finance, FRED)
- Pulls news directly from APIs (no expensive websearch)
- Writes Python scripts to calculate correlations, returns, statistics
- Re-prompts itself iteratively until the analysis is complete
- Generates markdown reports with findings

**Example:**
```
User: "Analyze AMD vs INTC correlation over 30 days"

Eli (autonomous loop):
  Iteration 1: Fetch AMD prices from Yahoo Finance
  Iteration 2: Fetch INTC prices from Yahoo Finance
  Iteration 3: Write Python to calculate correlation
  Iteration 4: Execute Python, get result (0.87)
  Iteration 5: Fetch news for both tickers
  Iteration 6: Generate markdown report
  Status: DONE
```

**No manual steps. Just ask the question.**

## How It Works

### 1. Native Rust Data Tools
**No websearch needed for data.**

Eli has built-in integrations written in Rust:
- **Yahoo Finance API** - Stock prices, volume, historical data
- **FRED API** - GDP, inflation, unemployment, treasury yields
- **News APIs** - Direct article fetching (no expensive LLM websearch)

These are **native Rust implementations**, not subprocess calls to Python scripts.

### 2. Autonomous Iteration Loop
**Eli re-prompts itself until done.**

Traditional AI chat: Ask question → Get one response → Done

Eli's approach:
```
User question
    ↓
Agent iteration 1: Plan the analysis
    ↓
Agent iteration 2: Fetch AMD data
    ↓
Agent iteration 3: Fetch INTC data
    ↓
Agent iteration 4: Write Python for correlation calculation
    ↓
Agent iteration 5: Execute Python, get result
    ↓
Agent iteration 6: Fetch news for context
    ↓
Agent iteration 7: Generate markdown report
    ↓
Status: DONE (return control to user)
```

**You don't guide it step-by-step. It works autonomously.**

### 3. Code Generation for Math
**Eli writes Python to do calculations on fetched data.**

Example: User asks for correlation analysis

Eli generates and executes:
```python
import numpy as np

# Calculate returns
amd_returns = np.log(amd_prices[1:] / amd_prices[:-1])
intc_returns = np.log(intc_prices[1:] / intc_prices[:-1])

# Pearson correlation
correlation = np.corrcoef(amd_returns, intc_returns)[0, 1]
print(f"Correlation: {correlation:.4f}")
```

The Python code is **generated by Eli based on the data fetched**, not hardcoded.

### 4. News Without Websearch
**Direct API calls, not LLM-powered search.**

Traditional AI agents use expensive websearch (Perplexity-style):
- Each news search costs LLM tokens
- Slow (multiple API round-trips)
- Rate-limited

Eli fetches news directly:
- Direct HTTP calls to news APIs
- Fast (no LLM involved)
- Free or cheap (standard REST APIs)

**Result:** Get context on stocks without expensive websearch.

## Why Rust?

### 1. Native Data Tools
**Yahoo Finance and FRED clients written in Rust.**

No subprocess calls. No Python dependencies. Direct HTTP/REST integration.

```rust
// Native Rust, not subprocess
let prices = yahoo_finance::fetch_ticker("AMD", "30d").await?;
let gdp = fred::fetch_series("GDP").await?;
```

Fast, type-safe, no overhead.

### 2. Parallel Execution
**Fetch 10 tickers simultaneously without data races.**

Rust's ownership system guarantees thread safety at compile time. Critical for parallel data fetching.

```rust
// Parallel fetching with tokio
let (amd, intc, nvda) = tokio::join!(
    fetch_ticker("AMD"),
    fetch_ticker("INTC"),
    fetch_ticker("NVDA")
);
```

### 3. Type Safety for Financial Data
**Prevent calculation errors at compile time.**

```rust
struct Price(f64);      // Price in dollars
struct Return(f64);     // Percentage return

// Compiler error if you try to add price + return
let x = Price(100.0) + Return(0.05);  // ❌ Won't compile
```

Financial calculations must be correct. Rust enforces this.

### 4. Single Binary Distribution
```bash
cargo build --release
# One binary, all tools included
./target/release/eli chat
```

No "install Python 3.11, pandas, numpy, yfinance, fredapi, requests..."

Just download the binary and run.

## Code Generation for Calculations

Eli **writes Python to do math on fetched data.**

### Example: Correlation Analysis

**User asks:** "Calculate AMD vs INTC correlation"

**Eli generates:**
```python
import numpy as np

# Data already fetched by Rust tools
amd_prices = [100.5, 101.2, 99.8, ...]  # From Yahoo Finance
intc_prices = [45.3, 45.7, 44.9, ...]   # From Yahoo Finance

# Calculate log returns
amd_returns = np.log(np.array(amd_prices[1:]) / np.array(amd_prices[:-1]))
intc_returns = np.log(np.array(intc_prices[1:]) / np.array(intc_prices[:-1]))

# Pearson correlation coefficient
correlation = np.corrcoef(amd_returns, intc_returns)[0, 1]

print(f"Correlation: {correlation:.4f}")
print(f"AMD avg return: {np.mean(amd_returns):.4f}")
print(f"INTC avg return: {np.mean(intc_returns):.4f}")
```

**Eli executes this Python, captures output, continues analysis.**

### Example: Divergence Detection

**User asks:** "Find divergence points between AMD and INTC"

**Eli generates:**
```python
import numpy as np

# Calculate standardized returns
amd_std = (amd_returns - np.mean(amd_returns)) / np.std(amd_returns)
intc_std = (intc_returns - np.mean(intc_returns)) / np.std(intc_returns)

# Find divergence (>2 standard deviations apart)
divergence_threshold = 2.0
divergence_points = []

for i in range(len(amd_std)):
    diff = abs(amd_std[i] - intc_std[i])
    if diff > divergence_threshold:
        divergence_points.append({
            'date': dates[i],
            'amd_return': amd_returns[i],
            'intc_return': intc_returns[i],
            'divergence': diff
        })

print(f"Found {len(divergence_points)} divergence points")
for point in divergence_points:
    print(f"  {point['date']}: AMD {point['amd_return']:.2%}, INTC {point['intc_return']:.2%}")
```

**Code is generated based on the specific question asked.**

## Design Philosophy

### 1. Autonomous, Not Manual
You shouldn't guide the AI step-by-step. Set it in motion and let it work.

### 2. Data First, Code Second
The value is instant access to financial data. Code is just the tool to analyze it.

### 3. Native Tools, Not Subprocess Wrappers
Yahoo Finance and FRED integrated in Rust. No Python subprocess calls for data.

### 4. Direct APIs, Not LLM Websearch
News from direct REST APIs. Fast and cheap. No expensive LLM-powered search.

### 5. Terminal Native
No browser. No Electron. Just your shell and a Rust binary.

### 6. Parallel by Default
Fetch 10 tickers simultaneously. Run 5 analyses in parallel. Use all CPU cores.

### 7. Reproducible Research
Every data fetch logged. Every iteration recorded. Replay any analysis.

### 8. Free & Accessible
Free LLM orchestration (OpenRouter). No signup required. Open source.
